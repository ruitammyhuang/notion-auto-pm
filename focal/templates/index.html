How to set this up (first time)


1
Go to notion.so/my-integrations and click + New integration. Give it a name (e.g. "WBS Sync"), set the workspace, and click Submit. Copy the Internal Integration Secret.
2
Paste the token into the Setup tab and click Save & Test.
3
Name every project WBS database starting with WBS — e.g. WBS - EDG 6648 Instruction. In Notion, open each WBS database → click ··· → Connections → add your integration. Do the same for the 📁 Projects and 📋 Master WBS Tasks databases.
4
Go to the Sources tab → click Discover WBS databases. Only databases starting with "WBS" appear — no filtering needed. Map each database's columns, choose its project (dropdown or paste page ID), and click Save configuration.
5
Go to the Sync tab → click Sync Now. Run again anytime you update your project WBS databases.

Why does my project not appear in the dropdown?


The dropdown only shows pages that are rows in the 📁 Projects database. If your project lives as a standalone page (e.g. inside a Teaching & Mentoring hub), it won't appear there.

Fix: use the "Or paste page ID" field below the dropdown. Open the project page in Notion, click ··· → Copy link, then extract the 32-character ID from the URL (the last segment before any ?). Paste it into the manual ID field — the dropdown selection is ignored when a manual ID is present.

What gets synced


A single Sync Now run does the full three-layer sync automatically:

Phase 1 — Project WBS → Master WBS Tasks. Syncs Task Name, Priority, Planned Start, Planned End, Notes, and Work Type. Status is set once on first create and never overwritten afterward, preserving Auto Status and any manual edits.

Phase 2 — Master WBS Tasks → Work Sessions. Creates one Work Session row per Master WBS task (Project + Task relation only). Tasks that already have a session are skipped — re-running is always safe.

Each task is tracked by its Notion page ID, so running sync again will update existing rows rather than create duplicates.

Back-link / Master WBS relation


After creating a new Master WBS entry, the tool automatically writes the page ID back into the source WBS row's Master WBS relation field (configurable per source). This keeps the two databases bi-directionally linked and makes the Auto Status rollup populate immediately without any manual linking.

Work Type override map


If your source WBS has a Category column instead of a Work Type column, paste a JSON map in the Work Type override field to translate automatically:
{"Weekly Announcement": "📣 Communication", "Grading": "✅ Review & Assessment"}

Valid Work Type values: ✍️ Writing, 🔍 Analysis, 📐 Design & Build, ✅ Review & Assessment, 🤝 Meeting, ⚙️ Admin, 📣 Communication.

Auto Planned Start


When enabled (default), if a task has no Planned Start but has a Due Date (Planned End), Planned Start is automatically set to Due Date − 7 calendar days. Uncheck per source if your WBS already has explicit start dates for every task.

Dynamic task deletion & renaming


When you delete or rename tasks in a project WBS, those changes flow through automatically on the next sync run:

Deleted task: The corresponding Master WBS entry is archived (moved to Notion Trash), and its linked Work Session is archived as well. Both are removed from the mapping files so they won't reappear on future syncs.

Renamed task: The Master WBS entry's Task Name is updated to match, and the linked Work Session's Session Name is updated too — keeping all three layers consistent with no manual work.

Note: Only tasks that were synced with the current version of the tool (v2 mappings format) participate in deletion detection. Legacy entries synced before this feature was added are left untouched until their next sync run, which will tag them automatically.

How status flows


There is no manual Status column anywhere. Status is entirely automatic:

Work Session (In Progress / Completed) → Master WBS Auto Status (formula, updates instantly) → Project WBS Auto Status (rollup via Master WBS relation, updates instantly).

To start a task: create a Work Session linked to the Master WBS entry. To complete it: mark the Work Session Completed. Both the master list and project WBS reflect the change immediately.
